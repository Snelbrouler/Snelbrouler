Introduction and Motivation

Gaming is one of the biggest industries in the world. Many gamers enjoy playing them, but many others find watching others play games equally as entertaining. Platforms such as Twitch let users livestream their games, allowing thousands of audience members to see their favorite internet personalities' reactions to their favorite games. Our goal is to find out how large streamers keep their fans watching their content. To do this, we'll be working with data from the top 1000 streamers of 2020. We'll be using the data science pipeline to find and analyze relationships that may be present between the popularity of a streamer and other factors regarding their stream and viewership.

Step 4: Machine Learning

For this step, we're going to use the variables we chose from our exploratory data analysis (Followers) to predict the watch time a Twitch streamer could get if they had a random set of values for the aforementioned variable.

For this part, we're going to want to use Scikit Learn to create a model for the data. In this specific example, we want to create a multiple regression model, since we have multiple different variables that we are treating as independent variables. In order to do so, we'll split our dataframe into randomly created subsets that will be used for either training or testing:

In[25]:

train, test = train_test_split(df, train_size = 0.2)

We can now create a linear model from the training data and see how accurate it is in predicting watch time. For the x training and testing set, the variables will be the same as the ones we picked out from before. The y training and testing set will use the dependent variable, watch time. We will then fit them into a linear regression model and allow it to make a prediction given our samples:

In[26]:

#we assign each training set to their respective columns in the dataframe
x_train = train[["Followers"]]
x_test = test[["Followers"]]
y_train = train["Watch time(Minutes)"]
y_test = test["Watch time(Minutes)"]
#this will create our linear regression model
reg = linear_model.LinearRegression()
reg.fit(x_train, y_train)
#now we will make predictions using our model on the randomly selected test data
y_train_pred = reg.predict(x_train)
y_test_pred = reg.predict(x_test)

After getting our predictions for both the training and test sets, it's time to get some regression metrics from the model so that we can gage how accurate it was in determining watch time. The metrics we'll be using are listed here on the Scikit Learn documentation: https://scikit-learn.org/stable/modules/model_evaluation.html#r2-score

We will only be calculating:

Explained Variance: the proportion to which our model accounts for the variation of a data set. The closer to 1, the better.

R-squared: how close the variables are to the regression model. Again, the closer the value is to 1 the better.

Let's see what these evaluate to using metrics from Scikit Learn:

In[27]:

explained_variance=metrics.explained_variance_score(y_train, y_train_pred)
print('Explained Variance: ', explained_variance)    
print('r squared: ', reg.score(x_train, y_train))

As mentioned before, we want both values to be as close to 1 as possible. However, with values as low as these, this is a clear indication that we are in need of a more refined model for calculating linear relationships and that just going by the number of followers a streamer has is not enough to determine their total watch time.

In our exploratory data analysis and in the heat map, we've observed that there are multiple different independent variables that also have a strong correlation with watch time besides just Followers. These include Peak viewers, Average viewers, Followers gained, and Views gained. So it seems that instead of using just a simple linear regression model for our data, we need to use a multiple regression model. That way, we are able to better explain the relationship between our single dependent variable and various other independent variables. Though it will be more difficult to visualize, it will hopefully allow us to create a better predictor for our values. So let's try again, this time by training x on multiple variables:

Now we need to test whether or not we should reject our null hypothesis, which states that there does not exist a relationship between peak viewers, average viewers, followers, followers gained, and views gained. We can do this by analyzing the relationship between the residuals for our new model (the distance between the actual plotted points and the linear regression model) and the total watch time in a histogram, which will show us the distribution of the differences between each point and the linear model:

In[33]:

Therefore, we can reject the null hypothesis, because this indicates a linear relationship between the set of independent variables and the dependent variable.

To do this for categorical data like the names of the languagaes spoken, we are going to have to use a dummy variable trap, which drops the first column of the list. This way, we can eliminate redundant information if we already know what the other values will be (for example, if English is a 0 in the list of dummy variables, then we know it must be one of the other categorical variables in the list). Then, we will need to fit a linear regression model for the interaction between the languages and watch time. The result is this:

Step 5: Making Conclusions

Despite the name of this step, we cannot make any clear conclusions based on the patterns we've seen in the data. We have proven that there does indeed exist a linear relationship between a streamer's total watch time and their followers, average viewers, peak viewers, followers gained, and viewers gained, though there is a lot we can still unpack when questioning the certainty of this statement. For example, we must also acknowledge that correlation doesn't mean causation; although the number of followers and watch time a streamer has have a linear relationship with each other, this does not prove that one causes the other or vice versa.

We've also demonstrated that there does exist a relationship between the language streamers speak and their popularity as well. For example, streamers who speak English, Portuguese, Spanish, or German tend to have the most watch time from audiences. Again, we cannot determine that one causes the other, for example, that a streamer who speaks Portuguese is going to have more watch time.

However, we can say for certain that with the methods we've used in our exploratory data analysis phase, although there does exist a linear relationship between watch time and other variables in the table, it is not that strong given the values we generated during our machine learning phase. More data is needed to make more concrete conclusions.

Final Thoughts and Resources

Twitch is a complex platform with thousands of creators playing thousands of different games. An issue with this project is that we've concluded that a streamer gets more total watch time if they already have a lot of followers and viewers, so using this data to determine how new streamers can grow their audiences would be ineffective.

If I were to do this again, I'd definitely want data on the games that streamers play, viewer demographics, and times throughout the year when a streamer experienced a significant increase in followers or views.
